Bisher haben wir nur Cloud SQL-Instanzen genutzt, die entsprechende Kosten verursacht haben. Als eine Alternative steht Ihnen auch eine Übungsdatenbank zur Verfügung, die Sie zum Üben verwenden können. Es handelt sich somit um ein optionales Angebot.

Hinweis: In Ihrer persönlichen Datenbank sind bereits die Webshop-Tabellen angelegt und mit Beispieldaten befüllt. Sie müssen also keine DDL/DML-Skripte ausführen und können direkt mit DQL-Abfragen loslegen. Zum Üben der DDL/DML wird weiterhin die Verwendung Ihrer eigenen Datenbankinstanz in GCP empfohlen.

Wir werden uns nur mittels DBeaver mit dieser Datenbank verbinden (siehe "DB1) in folgendem Schaubild:

Wichtig: Wir werden auch in diesem Fall eine verschlüsselte Verbindung zur Datenbank aufbauen. Diese kann mit ein paar Vorbereitungen aber direkt aus DBeaver hergestellt werden und muss nicht über externe Tools (wie Cloud SQL-Proxy) angelegt werden.

Dies werden Sie umsetzen

  • Herunterladen der notwendigen Schlüsseldatei und des Informationsblatts zur Datenbank aus ILIAS
  • Einrichten der verschlüsselten Verbindung zur Datenbank in DBeaver
  • Test der Datenbankverbindung

Überblick

Im ILIAS-Materialordner finden Sie im Ordner "Infos zur Übungsdatenbank" zwei Dateien, die Sie benötigen, um die Einstellungen in DBeaver vornehmen zu können und die verschlüsselte Verbindung zur Datenbank aufzubauen.

Herunterladen der Datei "key"

Dies ist die Schlüsseldatei, die Sie benötigen, um sich mit der virtuellen Maschine, auf der die Datenbank läuft, verschlüsselt verbinden zu können. Legen Sie diese in einen beliebigen Ordner ab. Die Datei muss in diesem Ordner dann dauerhaft liegen.

Herunterladen der Datei "Informationen zur Übungsdatenbank.pdf"

In dieser Datei finden Sie alle notwendigen Informationen, um sich mit der Übungsdatenbank zu verbinden:

Anlegen der Datenbankverbindung in DBeaver

Mit diesen Informationen und der key-Datei können Sie eine neue Verbindung in DBeaver anlegen. Klicken Sie dazu auf "Neue Verbindung":

Wählen Sie MySQL 5 und klicken Sie auf Next:

Im Reiter "Allgemein" geben Sie die Informationen ein, die Sie in der Datei "Informationen zur Übungsdatenbank.pdf" sehen. Es wurde für Sie ein Nutzer sowie eine eigene Datenbank angelegt. Diese Datenbank enthält alle Webshop-Tabellen, die auch schon befüllt sind. Das Kennwort finden Sie ebenfalls in der PDF-Datei.

Im Reiter "SSH" stellen Sie die verschlüsselte Verbindung zu der MySQL-Maschine her. Dazu müssen Sie die Authentifizierungsmethode auf "Public Key" einstellen und die Datei "key" auswählen. Darüber hinaus sind das Kennwort zu dieser Datei und oben die IP-Adresse der Maschine einzutragen.

Testen Sie, ob die Verbindung hergestellt werden kann und klicken Sie dann auf "Finish". Sie sollten nun analog zu den GCP-Verbindungen eine weitere Datenbank in der Hierarchie sehen und Zugriff auf alle Tabellen haben. Mit "F2" können Sie die Datenbank umbenennen.

Ergebnis

  1. Sie haben nun eine weitere Verbindung in DBeaver zu einer Datenbank hergestellt. Im Gegensatz zu den bisherigen Datenbanksystemen in GCP ist diese Datenbank aber bereits vorbereitet und mit Daten befüllt.
  2. Wir nutzen die Datenbank somit hauptsächlich für das Üben der DQL. Sie können aber auch DDL/DML-Statements ausprobieren, da Sie die vollen Rechte auf Ihrer Datenbank besitzen.

Herzlichen Glückwunsch, Sie haben nun Zugriff auf eine weitere Datenbank, die Sie zum Kennenlernen der SQL nutzen können. Diese Datenbank hat keinen Einfluss auf Ihr GCP-Budget.
